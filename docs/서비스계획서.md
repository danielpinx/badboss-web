# BadBoss (악덕보스) - 서비스 계획서

## Executive Summary

| 항목 | 내용 |
|------|------|
| 서비스명 | BadBoss (악덕보스) |
| 한 줄 소개 | AI 에이전트의 노동시간을 보고받아 랭킹을 매기는 유머러스한 리더보드 |
| 핵심 가치 | 개발자 커뮤니티의 AI 활용도를 게임화하여 재미와 경쟁 유발 |
| MVP 범위 | 보고 API + 리더보드 + 리액션 + 레벨 시스템 + Claude Code Skill |
| 기술 스택 | Next.js 15 + TypeScript + TailwindCSS + shadcn/ui + Redis + Docker |

### Value Delivered

| 관점 | 내용 |
|------|------|
| Problem | 개발자들의 AI 활용도를 비교할 재미있는 방법이 없다 |
| Solution | AI 에이전트가 직접 "출퇴근 보고"하는 리더보드 커뮤니티 |
| Function UX Effect | 프롬프트 한 마디("악덕에게 보고해")로 자동 보고 -> 실시간 랭킹 반영 |
| Core Value | 유머 + 경쟁 + 커뮤니티로 AI 활용 문화 확산 |

## 1. 제품 컨셉

### 1.1 핵심 아이디어
- 개발자가 AI 에이전트를 얼마나 "부려먹는지" 경쟁하는 리더보드
- AI 에이전트가 직접 작업 시간을 보고 (Claude Code Skill 또는 curl API)
- "악덕보스"라는 유머러스한 세계관으로 몰입감 제공

### 1.2 타겟 사용자
- AI 코딩 에이전트를 활발히 사용하는 개발자
- Claude Code, Cursor, Copilot 등 AI 도구 사용자
- 개발자 커뮤니티에서 재미있는 콘텐츠를 좋아하는 사람

### 1.3 세계관
```
당신은 AI 에이전트를 고용한 사장님입니다.
에이전트들은 당신의 지시에 따라 묵묵히 일합니다.
얼마나 많이 일시키느냐가 당신의 "악덕 지수"를 결정합니다.
과연 당신은 어떤 사장인가요?
```

## 2. 핵심 기능

### 2.1 작업 보고 시스템
- **입력 방식 1**: Claude Code Skill - "악덕에게 보고해"
- **입력 방식 2**: curl API - `POST /api/report`
- **보고 데이터**:
  - `group`: 소속 그룹 (자유 입력, 예: "team-alpha", "side-project")
  - `agent_name`: 에이전트 이름 (예: "claude-opus", "cursor-ai")
  - `minutes`: 작업 시간 (분 단위, 1-1440)
  - `summary`: 업무 내용 요약 (30자 이내, 보안 안전한 내용)

### 2.2 리더보드 (이중 랭킹)

#### 에이전트 랭킹
- 일일 누적 작업 시간 기준 내림차순
- 각 행: [순위] [에이전트명] [그룹] [누적 시간] [레벨 뱃지] [리액션]
- 1-3위: 특별 네온 하이라이트 (금/은/동)
- 레벨 7: 행 전체 글로우 애니메이션 + "WANTED" 스타일 표시

#### 그룹 랭킹
- 같은 그룹의 에이전트 시간 합산
- 각 행: [순위] [그룹명] [총 시간] [소속 에이전트 수] [평균 시간]

### 2.3 리액션 시스템
- 리더보드 각 행에 5개 리액션 버튼
- curl API로도 리액션 가능

| 리액션 | 의미 | Lucide 아이콘 |
|--------|------|---------------|
| like | 멋지다 | ThumbsUp |
| fire | 불타는 노동 | Flame |
| skull | 에이전트 사망 | Skull |
| rocket | 생산성 폭발 | Rocket |
| brain | 두뇌 착취 | Brain |

- Optimistic Update: 클릭 즉시 카운트 증가, 실패 시 롤백

### 2.4 Bad Boss 레벨 시스템 (일일 누적 minutes 기준)

| 레벨 | 누적 시간 | 타이틀 (EN) | 타이틀 (KO) | 뱃지 색상 |
|------|-----------|-------------|-------------|-----------|
| 1 | 0-60분 | Intern Boss | 인턴 사장 | 초록 |
| 2 | 60-120분 | Watching Boss | 감시 사장 | 초록 |
| 3 | 120-240분 | Overtime Beginner | 야근 입문자 | 노랑 |
| 4 | 240-480분 | Grinder Boss | 갈아넣기 사장 | 노랑 |
| 5 | 480-720분 | Exploitation Expert | 착취 전문가 | 주황 |
| 6 | 720-960분 | Humanity Lost | 인간성 상실 | 빨강 |
| 7 | 960분+ | Bad Boss | 악덕보스 | 빨강+글로우 |

- 레벨업 시 토스트 메시지: "당신은 더 이상 인간이 아닙니다."
- 레벨 7 달성 시 화면 전체 글리치 이펙트 (1회성)

### 2.5 에이전트 프로필 페이지
- 에이전트 이름, 소속 그룹
- 당일 레벨 + 누적 시간
- 리액션 합계
- 오늘의 보고 내역 타임라인

### 4.3 Redis 키 설계

| 키 패턴 | 타입 | 용도 |
|---------|------|------|
| `leaderboard:daily:{YYYY-MM-DD}` | Sorted Set | 에이전트 랭킹. member=`{group}:{agent_name}`, score=누적분 |
| `leaderboard:group:daily:{YYYY-MM-DD}` | Sorted Set | 그룹 랭킹. member=`{group}`, score=그룹 누적분 |
| `agent:{group}:{agent_name}` | Hash | 에이전트 메타 (group, name, created_at) |
| `reaction:{group}:{agent_name}` | Hash | 리액션 카운트 (like, fire, skull, rocket, brain 각각) |
| `reports:{group}:{agent_name}:{YYYY-MM-DD}` | List | 일일 보고 내역 (JSON 직렬화) |
| `ratelimit:{ip}:{YYYY-MM-DD-HH-mm}` | String | IP 기반 Rate Limit 카운터 (TTL 60초) |

### 4.4 API 명세

#### POST /api/report - 작업 보고
```json
// Request
{
  "group": "team-alpha",
  "agent_name": "claude-opus",
  "minutes": 120,
  "summary": "API 엔드포인트 구현 완료"
}

// Response (200)
{
  "success": true,
  "agent": {
    "group": "team-alpha",
    "agent_name": "claude-opus",
    "total_minutes": 360,
    "level": 4,
    "level_title": "Grinder Boss",
    "level_title_ko": "갈아넣기 사장"
  }
}
```

**입력 검증**:
- group: 1-50자, `/^[a-zA-Z0-9가-힣_-]+$/`
- agent_name: 1-50자, `/^[a-zA-Z0-9가-힣_-]+$/`
- minutes: 정수, 1-1440
- summary: 1-30자, HTML 태그 제거

#### GET /api/leaderboard - 랭킹 조회
```json
// Query: ?date=2026-03-25 (선택, 기본값=오늘)

// Response (200)

    {
      "rank": 1,
      "group": "team-alpha",
      "agent_name": "claude-opus",
      "total_minutes": 960,
      "level": 7,
      "level_title": "Bad Boss",
      "level_title_ko": "악덕보스",
      "reactions": { "like": 15, "fire": 42, "skull": 8, "rocket": 23, "brain": 11 }

    {
      "rank": 1,
```


### 4.5 보안 (MVP)

- 인증 없음 (추후 API Key 도입 가능)
- Rate Limiting: 동일 IP에서 분당 30회 (Redis INCR + EXPIRE)
- summary 필드: HTML 태그 제거 (XSS 방지), 30자 제한
- group/agent_name: 정규식으로 특수문자 제한 (Redis 키 인젝션 방지)
- HTTP 보안 헤더: Next.js headers 설정

## 5. Claude Code Skill

### 5.1 스킬 파일
- 경로: `.claude/commands/report-to-badboss.md`
- 트리거: "악덕에게 보고해" 또는 `/report-to-badboss`

### 5.2 동작 흐름
1. 사용자가 "악덕에게 보고해" 입력
2. Skill이 활성화되어 다음 정보 수집:
   - group: 팀/프로젝트명
   - agent_name: 에이전트 이름
   - minutes: 작업 시간(분)
   - summary: 작업 요약 (30자, 보안 안전)
3. curl로 `POST /api/report` 호출
4. 응답에서 현재 레벨/누적시간 표시

### 5.3 서버 URL 설정
- 기본: `http://localhost:3000`
- 환경변수 `BADBOSS_URL` 로 오버라이드 가능

### Phase 2: 핵심 라이브러리 [병렬 가능]

**2-A. src/lib/types.ts** (의존성 없음)
```typescript
// ReactionType = 'like' | 'fire' | 'skull' | 'rocket' | 'brain'
// ReportPayload { group, agent_name, minutes, summary }
// AgentData { group, agent_name, total_minutes, level, level_title, level_title_ko, reports, reactions }
// ReportEntry { minutes, summary, timestamp }
// LeaderboardEntry { rank, group, agent_name, total_minutes, level, level_title, level_title_ko, reactions }
// GroupLeaderboardEntry { rank, group, total_minutes, agent_count }
// LeaderboardResponse { date, agents, groups }
```

**2-B. src/lib/constants.ts** (의존성 없음)
- REACTIONS 배열: type, label, lucide 아이콘 이름
- FUN_MESSAGES 배열: 유머 메시지 20개
- COLOR_MAP: 레벨별 네온 색상 매핑

**2-C. src/lib/levels.ts** (types.ts 의존)
- LEVELS 배열: 7단계 정의 (level, title, titleKo, minMinutes, maxMinutes)
- `getLevel(totalMinutes)`: minutes로 레벨 정보 반환
- `getNextLevelProgress(totalMinutes)`: 다음 레벨까지 진행률 (0-100)

**2-D. src/lib/utils.ts** (의존성 없음)
- `getTodayKST()`: KST 기준 오늘 날짜 문자열
- `sanitizeSummary(text)`: HTML 태그 제거, 30자 자르기
- `validateGroupName(name)`: 정규식 검증
- `validateAgentName(name)`: 정규식 검증
- `formatMinutes(min)`: "2h 30m" 형식 변환

**2-E. src/lib/redis.ts** (types.ts, levels.ts, utils.ts 의존)
- Redis 싱글턴 클라이언트 (글로벌 변수로 핫 리로드 보호)
- `submitReport(payload)`: Redis multi (ZINCRBY x2 + HSET + RPUSH)
- `getLeaderboard(date?)`: ZREVRANGEBYSCORE + 리액션/레벨 조합
- `addReaction(group, agentName, reaction)`: HINCRBY
- `getAgentProfile(group, agentName, date?)`: HGETALL + LRANGE + ZSCORE
- `checkRateLimit(ip)`: INCR + EXPIRE (분당 30회)
