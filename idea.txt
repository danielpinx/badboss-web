You are a senior full-stack engineer and product builder.

Build a production-ready web service called "badboss (악덕대표)".

---

# 1. PRODUCT OVERVIEW

## Goal
A fun, competitive leaderboard-based community where developers compare how intensively they use AI agents (LLMs, coding agents, etc).

This service is intentionally humorous and slightly provocative:
Users are “competing” on how hard they “make their AI work”.

---

## Core Concept
- Users submit daily AI usage time (in hours)
- Leaderboard ranks users by usage time
- Users can react to others via fun icons (also via curl API)
- Gamified “Bad Boss Levels” based on usage time

---

# 2. CORE FEATURES (MVP)

## 2.1 Leaderboard
- Daily leaderboard (reset every day)
- Sorted by usage hours (descending)
- Top users displayed prominently

---

## 2.2 AI Usage Submission
- Users submit:
  - user_id
  - hours (integer or float)

---

## 2.3 Reaction System (IMPORTANT)
- Each leaderboard row has reaction buttons:
  - like 👍
  - fire 🔥

# 4. REDIS DATA DESIGN

Use clear and scalable structure.

## Leaderboard
Key:
leaderboard:daily:{YYYY-MM-DD}

Type:
Sorted Set

Value:
ZADD leaderboard:daily:{date} {hours} {user_id}

---

## User Data
Key:
user:{user_id}

Fields:
- nickname
- today_hours
- level

---

## Reactions
Key:
reaction:{target_user_id}

Fields:
- like
- fire
- skull
- rocket
- brain

---

# 5. API SPEC

## 5.1 Submit Time
POST /api/time

Request:
{
  "user_id": "daniel",
  "hours": 14
}

Behavior:
- update leaderboard
- update user level

---

## 5.2 Get Leaderboard
GET /api/leaderboard

Response:
- sorted users
- include:
  - user_id
  - hours
  - level
  - reactions

---

## 5.3 React (CRITICAL - curl support)
POST /api/react

Request:
{
  "target_user_id": "daniel",
  "reaction": "fire"
}

Behavior:
- increment reaction count in Redis

---

## 5.4 Example curl
curl -X POST http://localhost:3000/api/react 
  -H "Content-Type: application/json" 
  -d '{"target_user_id":"daniel","reaction":"fire"}'
